          同城数据采集 — 功能说明与使用教程
═══════════════════════════════════════════════════════════════

【背景】

抖音、微信视频号等平台对自动化有严格的反检测机制（验证码、
IP风控、接口签名等）。为解决「同城数据采集」需求，我们采用
了 地理位置伪装（GPS Spoofing）+ 浏览器会话缓存 方案：

  · 通过 Playwright 浏览器向平台伪造 GPS 坐标为目标城市
  · 平台的推荐算法会根据位置推送附近/同城内容
  · 首次验证通过后，会话被缓存，后续采集自动复用，无需重复验证

采集过程中，界面底部状态栏会实时显示当前状态，每轮采集开始
时会显示轮次标识（如"第 1 轮采集开始"），不同平台顺序执行，
状态信息不会混淆。

───────────────────────────────────────────────────────────────
一、抖音采集策略（2 个方案 + 兜底）
───────────────────────────────────────────────────────────────

  方案1: 地理位置伪装采集（自动）
  ├─ 原理：创建浏览器并将 GPS 坐标设为目标城市（如深圳：
  │        纬度 22.54, 经度 114.06），抖音页面的 JavaScript
  │        读取到伪装位置后，向服务器请求附近内容
  ├─ 模式：无头浏览器（不弹窗），速度快
  ├─ 优点：全自动，无需任何操作
  └─ 注意：如果遇到验证码会自动跳到方案2

  方案2: 移动端模拟采集（手动验证）
  ├─ 原理：模拟手机浏览器（Android Chrome）+ GPS 伪装，
  │        以有头模式打开浏览器，用户可手动完成验证码
  ├─ 模式：弹出可见的浏览器窗口
  ├─ 用户操作：
  │    1. 如弹出图案验证码 → 手动拖拽完成
  │    2. 如要求输入手机号/验证码 → 手动输入
  │    3. 完成后工具自动继续采集
  └─ 重要：验证通过后 Cookie 会被保存！下一轮采集时方案1
           会自动加载这些 Cookie，通常不会再触发验证码。
           也就是说：验证码只需要过一次。

  兜底: 主页信息流
  ├─ 当上述方案均未成功时自动触发
  ├─ 从抖音首页推荐流中采集内容
  └─ 注意：此内容为全国推荐，非同城定向数据

  ★ 会话缓存机制 ★
  方案1成功后，浏览器会话（页面、Cookie）会被保留在内存中。
  下一轮采集时直接刷新该页面即可获取新数据，无需重新打开
  浏览器、无需重新登录、无需重新过验证码。只有在会话失效时
  才会重新创建。

───────────────────────────────────────────────────────────────
二、微信视频号采集策略（3 个方案）
───────────────────────────────────────────────────────────────

  方案1: 隐身模式
  ├─ 使用反检测补丁隐藏自动化特征
  └─ 全自动

  方案2: 持久化浏览器
  ├─ 复用浏览器配置文件，保留登录状态
  └─ 全自动

  方案3: 标准模式兜底
  ├─ 使用默认浏览器连接
  └─ 全自动

  提示：微信视频号需要扫码登录。请先在「设置 → 平台登录」
  中完成微信视频号的登录。所有方案尝试完毕后会明确提示
  「本轮未获取到数据」，不会与其他平台的结果混淆。

───────────────────────────────────────────────────────────────
三、使用步骤
───────────────────────────────────────────────────────────────

  1. 打开工具，登录（默认：admin / admin123）

  2. 进入「设置」：
     ·「目标城市」— 填写城市名（支持 50+ 城市，如：深圳、
       北京、上海、成都、杭州、武汉、重庆 等）
     · 各平台「登录」— 点击登录按钮，在弹出的浏览器中完成
       登录（扫码或账号密码），关闭浏览器即可
     ·「大模型配置」— 如需情感分析，填写 API 地址和 Key
     ·「机器人配置」— 如需告警推送，填写钉钉/企微 Webhook

  3. 回到「采集」页面：
     · 勾选要采集的平台
     · 点击「开始采集」

  4. 观察状态栏信息（平台顺序执行，不会混淆）：
     · "═══ 第 1 轮采集开始 ═══"
     · "[抖音] 正在尝试 方案1: 地理位置伪装采集(自动)..."
     · "[抖音] 方案1 成功！获取 25 条同城数据"
     · "[微信视频号] 正在尝试 方案1: 隐身模式..."
     · ...

  5. 如果弹出浏览器窗口（方案2）：
     · 完成图案验证码（拖拽图案到指定位置）
     · 如需输入手机号和验证码，正常输入
     · 完成后工具自动继续，Cookie 自动保存
     · 下一轮采集通常不会再要求验证

  6. 采集完成后，可在「分析」页面进行情感分析，或「导出」数据

───────────────────────────────────────────────────────────────
四、技术措施说明
───────────────────────────────────────────────────────────────

  · GPS 地理位置伪装
    内置 50+ 城市的精确 GPS 坐标，通过浏览器 Geolocation API
    向平台伪装用户所在位置，使推荐算法返回目标城市附近内容

  · 移动端模拟
    模拟 Android 手机浏览器（Pixel 7 + Chrome），触发平台的
    移动端内容展示逻辑，移动端更容易出现同城/附近标签

  · playwright-stealth 反检测补丁
    隐藏 navigator.webdriver 等自动化痕迹

  · 随机浏览器指纹
    每次采集随机选择 User-Agent、窗口分辨率等参数

  · 拟人化操作
    模拟真人的滚动速度和随机停顿

  · Cookie 持久化 + 会话缓存
    登录后保存 Cookie，成功的浏览器会话在内存中缓存复用，
    避免重复登录和验证码。这是解决「重复验证」问题的核心

  · 平台顺序执行
    各平台按顺序采集（非并行），状态信息清晰，不会出现
    微信视频号采集时显示抖音数据的混淆情况

  · 轮次标识
    每轮采集开始时显示明确的轮次标记，便于区分不同采集周期

六、常见问题
───────────────────────────────────────────────────────────────

  Q: 还是需要每次都过验证码吗？
  A: 不需要。首次采集可能需要在方案2中手动过一次验证码，
     之后 Cookie 会被保存，后续采集方案1 会自动使用这些
     Cookie，通常不会再触发验证码。如果长时间后 Cookie
     过期，可能需要再过一次。

  Q: 采集到的内容是同城的吗？
  A: 通过 GPS 伪装，平台会根据伪装的位置推送附近内容。
     这比之前的关键词搜索（如搜索"深圳"）更接近真实的
     同城推荐。如果页面上有同城/附近标签，工具会自动点击。

  Q: 采集时提示"需安装 Chrome 或 Edge"？
  A: 请确认电脑已安装 Google Chrome 或 Microsoft Edge。

  Q: 微信视频号采集显示了抖音的内容？
  A: 已修复。现在各平台顺序执行，每个平台的结果独立显示。
     如仍遇到此问题，请确认使用的是最新版本。

  Q: 所有方案都失败了怎么办？
  A: ① 先在「设置」中完成平台登录
     ② 检查网络连接
     ③ 等待一段时间后重试（风控有时效性）

  Q: 我的城市不在支持列表中？
  A: 请联系技术支持添加，只需提供城市名和大致坐标即可。
